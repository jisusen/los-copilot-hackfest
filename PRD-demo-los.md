**For:** Claude Code  
**Project:** Credit Analyst Copilot — Demo Environment  
**Stack:** Bun + React + SQLite (bun:sqlite)  
**Port:** 3001  
**Last Updated:** April 2025

---

## 0. Mission Statement

Build a **realistic-looking internal banking web app** (Loan Origination System) for Indonesian personal loans. This is the "victim" app that the Browser Use AI agents will autonomously navigate during the hackfest demo. It must look like a real enterprise banking tool, be fast to spin up, and have predictable HTML structure so agents can reliably extract data.

**This is NOT a toy.** It needs to look like the real thing — proper banking UI, Indonesian terminology, realistic data. The demo lives or dies on how convincing this LOS looks.

## 1. Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Runtime | **Bun** | Fast install, built-in SQLite, single binary |
| Frontend | **React 18** (via Bun) | Component model, good for tab UI |
| Routing | **React Router v6** | Client-side routing for tab navigation |
| Styling | **Tailwind CSS v3** | Utility-first, fast to build enterprise UI |
| Database | **SQLite via `bun:sqlite`** | Zero config, file-based, easy to seed/reset |
| API | **Bun HTTP server** | Built-in, no Express needed |
| Bundler | **Bun bundler** | Native, no Vite/Webpack config overhead |

## 2. Visual Design Requirements

### Aesthetic Direction
**Enterprise banking system, circa 2022.** Think: Indonesian state-owned bank internal tool. Dense information layout, conservative color palette, functional typography. This is NOT a modern SaaS app — it should look like software built by an IT department, not a startup. That authenticity is what makes the demo convincing.

### Color Palette
```css
--color-primary:     #1a3a5c;   /* Deep navy — header, sidebar, buttons */
--color-primary-light: #2d5a8e; /* Lighter navy — hover states */
--color-accent:      #e8a020;   /* Amber/gold — badges, highlights */
--color-bg:          #f4f6f8;   /* Light grey page background */
--color-surface:     #ffffff;   /* Cards, panels */
--color-border:      #d1d9e0;   /* Table borders, dividers */
--color-text:        #1f2d3d;   /* Primary text */
--color-text-muted:  #6b7c93;   /* Labels, secondary text */
--color-success:     #1a7f4b;   /* Approve / Lancar */
--color-warning:     #c47d0e;   /* Review / DPK */
--color-danger:      #b91c1c;   /* Reject / KL-Macet */
```

### Typography
- **Display / Header:** `DM Sans` (Google Fonts) — clean, slightly formal
- **Body / Data:** `IBM Plex Sans` (Google Fonts) — technical, readable at small sizes
- **Monospace (ID numbers, amounts):** `IBM Plex Mono` — for NIK, NPWP, currency values

### Layout Principles
- Fixed sidebar (220px) on desktop — shows bank logo + nav
- Fixed topbar (56px) — shows current user + bank name
- Content area: scrollable, max-width 1200px
- Tab navigation: horizontal tabs directly under page title
- Data fields: two-column grid layout (label left, value right)
- Tables: alternating row colors, compact padding
- Dense information display — this is enterprise, not consumer

### Bank Identity (Fictional)
- **Bank name:** `Bank Mitra Sejahtera` (abbreviated: `BMS`)
- **Logo:** Simple text logo — `BMS` in a navy rounded square
- **Tagline:** `Sistem Informasi Kredit Konsumer`
- Use this consistently across all pages

## 5. Database Schema

### File: `server/db/schema.sql`

```sql
CREATE TABLE IF NOT EXISTS loan_applications (
    id TEXT PRIMARY KEY,
    status TEXT NOT NULL DEFAULT 'pending',
    created_at TEXT NOT NULL,
    analyst_id TEXT,
    product_type TEXT NOT NULL,
    amount_requested REAL NOT NULL,
    tenor_months INTEGER NOT NULL,
    loan_purpose TEXT,
    branch TEXT,
    marketing_officer TEXT,
    interest_rate REAL
);

CREATE TABLE IF NOT EXISTS debtors (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    full_name TEXT NOT NULL,
    nik TEXT NOT NULL,
    npwp TEXT,
    date_of_birth TEXT,
    marital_status TEXT,
    dependents INTEGER DEFAULT 0,
    employment_type TEXT,
    employer_name TEXT,
    job_title TEXT,
    years_employed REAL,
    domicile_city TEXT,
    domicile_address TEXT,
    phone TEXT,
    email TEXT
);

CREATE TABLE IF NOT EXISTS financials (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    gross_income REAL,
    net_income REAL,
    existing_obligations REAL DEFAULT 0,
    requested_installment REAL,
    total_obligations REAL,
    dti_ratio REAL,
    dti_threshold REAL,
    remaining_income REAL,
    income_verified INTEGER DEFAULT 0,
    verification_docs TEXT
);

CREATE TABLE IF NOT EXISTS slik_ojk (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    check_date TEXT,
    kolektibilitas INTEGER,
    kolektibilitas_label TEXT,
    worst_kol_12m INTEGER,
    payment_history_24m TEXT,
    existing_bank TEXT,
    existing_facility TEXT,
    existing_amount REAL DEFAULT 0,
    total_obligations_slik REAL DEFAULT 0,
    blacklist_status INTEGER DEFAULT 0,
    notes TEXT
);

CREATE TABLE IF NOT EXISTS aml_fraud (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    screening_date TEXT,
    dttot_match INTEGER DEFAULT 0,
    un_sanctions_match INTEGER DEFAULT 0,
    pep_status INTEGER DEFAULT 0,
    pep_detail TEXT,
    income_consistent INTEGER DEFAULT 1,
    address_flag INTEGER DEFAULT 0,
    fraud_signals TEXT,
    notes TEXT,
    engine_version TEXT DEFAULT 'BMS AML Engine v2.3'
);

CREATE TABLE IF NOT EXISTS crde_results (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    processed_at TEXT,
    risk_score TEXT NOT NULL,
    decision TEXT NOT NULL,
    numeric_score INTEGER,
    dti_actual REAL,
    dti_threshold REAL,
    dti_passed INTEGER,
    kol_value INTEGER,
    kol_passed INTEGER,
    aml_passed INTEGER,
    fraud_passed INTEGER,
    rules_triggered TEXT,
    notes TEXT,
    engine_version TEXT DEFAULT 'BMS CRDE v3.1.0'
);

CREATE TABLE IF NOT EXISTS collaterals (
    id TEXT PRIMARY KEY,
    loan_id TEXT NOT NULL REFERENCES loan_applications(id),
    required INTEGER DEFAULT 0,
    asset_type TEXT,
    asset_description TEXT,
    market_value REAL,
    liquidation_value REAL,
    appraisal_date TEXT,
    ltv_ratio REAL,
    ltv_threshold REAL,
    certificate_number TEXT,
    legal_status TEXT,
    notes TEXT
);

CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    created_at TEXT NOT NULL,
    expires_at TEXT NOT NULL
);
```

## 7. API Endpoints

All endpoints return JSON. All require session auth except `/api/auth/login`.

### Authentication

```
POST /api/auth/login
  Body: { username: string, password: string }
  Returns: { ok: true, user: { username, role } }
  Sets: HttpOnly session cookie

POST /api/auth/logout
  Returns: { ok: true }
  Clears: session cookie

GET /api/auth/me
  Returns: { user: { username, role } } or 401
```

### Loans

```
GET /api/loans
  Query: ?status=pending&product=KTA&search=budi
  Returns: { loans: LoanSummary[], total: number }

GET /api/loans/:id
  Returns: { loan: LoanDetail }
  LoanDetail includes all joined tables:
  {
    application: LoanApplication,
    debtor: Debtor,
    financials: Financials,
    slik: SLIKResult,
    amlFraud: AMLFraudResult,
    crde: CRDEResult,
    collateral: Collateral | null
  }
```

### Admin / Dev

```
POST /api/admin/seed
  Resets and re-seeds the database
  Returns: { ok: true, count: number }
  (Protected — only in development mode)
```

tab-content-hasil-crde
tab-content-agunan

# Special

### Agent Navigation Rules (document these in README)

1. After login, always wait for `loan-queue-table` to be visible before clicking any loan
2. After clicking a tab, always wait for the corresponding `tab-content-{tab-name}` to be visible
3. All `value-*` fields are plain text spans — use `innerText` to extract
4. Currency values are formatted as "Rp 50.000.000" — strip non-numeric for calculations
5. Boolean flags (blacklist, pep, etc.) are rendered as "✅ Tidak Terdaftar" or "⚠️ TERDAFTAR"
6. The CRDE decision is always one of: "DISETUJUI", "PERLU REVIEW KOMITE", "DITOLAK"
